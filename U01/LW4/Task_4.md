Вывод:
Данная задача показывает работу индекса Index Range Scan.

Рассмотрим план выполнения запроса. В нашем случае применен метод INDEX RANGE SCAN. 
Он применяется когда в предикат where входят столбцы индекса (not UNIQUE) с условиями 
= (в случае если индексированные значения неуникальны), >, <, а также like ‘pattern%’, 
причем wildcard-символы должны стоять после искомой подстроки. Что полностью соотверствует нашим условиям.
По найденому значению индекса T2_IND_ID определяются blocks соотверствуещие ему и записи в них, 
после чего они выводятся.
